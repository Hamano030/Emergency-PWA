山などの電波が届かない場所で作業中に怪我をした場合に、衛星経由のSMS機能を使って事務所の担当職員に連絡するための緊急連絡アプリです。

**連絡の流れ**: 現場 → 事務所の担当職員(携帯電話) → 職員が119番通報

現場から直接119番へ連絡するのではなく、まず事務所の職員へSMSを送り、状況を伝えた職員が119番へ通報する、という2段階の流れになっています。

## アプリ製作の背景


山で作業中に怪我をした場合、通常の電波があればその場で119番に電話すればよい
問題は**通常の電波が届かない場所**での怪我
スマホの衛星通信サービス(docomo Starlink Directなど)を使えば、電波が圏外でも**SMSだけは送信できる**
ただし、衛星経由の「データ通信(Webページの閲覧など)」は対応アプリ限定・条件付きであり、圏外になってから新しくWebページを開くのは不確実
そのため、**電波がある間にあらかじめアプリをホーム画面に追加してキャッシュしておき**、山の中で電波が無くなっても画面自体はオフラインで開けるようにしてある
ボタンを押すと、**事前に登録しておいた事務所担当職員の携帯番号宛て**のSMS送信画面が開き、そのSMS送信自体は衛星通信を使って圏外でも送れる、という仕組みを想定している
SMSを受け取った事務所の職員が、現場の状況を確認したうえで119番へ通報する


**重要**: このアプリは「圏外・データ通信不可・SMSのみ生きている」という状況で使うことを前提に作られています。

## 事前準備(必ず電波がある場所で行う)

このアプリは**山に入る前**、まだ通常の電波がある自宅や登山口などで準備しておく必要があります。圏外になってから初めてアクセスしても正しく動作しません。

### スマホのホーム画面に追加する方法

**iPhone(Safari)の場合**

電波のある場所で、上記URLをSafariで開く
画面下の「共有」ボタンをタップ
メニューから「ホーム画面に追加」を選択
名前を確認して「追加」をタップ


**Androidの場合**

電波のある場所で、上記URLをChromeで開く
右上の「⋮」メニューをタップ
「ホーム画面に追加」または「アプリをインストール」を選択


これで、アプリの画面(HTML・アイコン・設定ファイル)がすべて端末内にキャッシュ(保存)され、以降は電波が無い場所でもアプリを開けるようになります。

### 衛星SMS機能が使える状態か確認しておく


使用するスマホが衛星通信(docomo Starlink Directなど)対応機種か確認する
対応機種であれば、契約プランに関わらず申し込み不要・無料で利用可能な場合が多い
事前に、圏外時にSMSが送受信できる状態になっているかテストしておくことを推奨

## 定期的な動作確認(重要)

一度ホーム画面に追加しても、以下のような操作でキャッシュが消え、**いざという時にアプリが開けなくなる**ことがあります。

**キャッシュが消える主なケース**

ブラウザの「履歴とWebサイトデータを消去」を実行した
設定アプリからWebサイトデータを個別に削除した
スマホの空き容量が少なく、OSが自動的にキャッシュを削除した
ホーム画面のアイコンを削除し、再登録し忘れた
OS・ブラウザの大型アップデート時にリセットされた


**確認方法(月1回を目安に推奨)**

スマホを機内モードにする(電波を完全にオフにする)
ホーム画面のアイコンをタップしてアプリを開く
問題なく画面が表示されるか確認する
表示されない場合は、電波のある場所で「事前準備」の手順(ホーム画面への追加)をやり直す


### 万が一のためのバックアップ手段

アプリだけに頼らず、以下も併せて備えておくことを推奨します。

緊急連絡先の電話番号を、スマホの連絡先アプリにも別途保存しておく
送信したいSMSの文面(定型文)を、メモアプリなどにも保存しておく
山に入るメンバー間で、この確認作業を共有しておく

## 圏外での使い方


怪我をした場合、ホーム画面に追加したアイコンをタップしてアプリを開く(電波が無くても、キャッシュ済みなのでそのまま開く)
アプリ内のボタンをタップし、**事務所の担当職員**へのSMS送信画面を呼び出す
送信する(空が見える屋外であれば、衛星経由でSMSが送信される)
SMSを受け取った事務所の職員が、状況を確認したうえで119番へ通報する

## ファイル構成(開発者向け)

| ファイル名 | 役割 |
|---|---|
| `index.html` | アプリ本体のHTML |
| `manifest.json` | PWAの設定ファイル(名前・アイコン・起動方法などを定義) |
| `service-worker.js` | オフライン動作のためのキャッシュ管理スクリプト |
| `icon-192.png` | ホーム画面アイコン(192×192) |
| `icon-512.png` | ホーム画面アイコン(512×512、高解像度端末用) |

### service-worker.jsのキャッシュ対象について

このアプリは「電波ゼロの状態で完全に動くこと」が前提のため、`urlsToCache`には**画面表示に必要な全ファイル**を含める必要があります。アイコン画像を含め忘れると、オフライン時に表示が一部崩れる可能性があるため注意してください。

```js
const urlsToCache = [
  "./",
  "./index.html",
  "./manifest.json",
  "./icon-192.png",
  "./icon-512.png"
];
```

## 連絡先(事務所職員の電話番号)を変更したい場合

事務所の担当者が変わった場合など、SMS送信先の電話番号を修正する必要があります。


`index.html` 内にある送信先電話番号の記述を探して修正する(SMSリンクの電話番号部分)
修正後、必ずローカルで動作確認してからGitHubにpushする
pushしたら、**全員に「ホーム画面から削除→再登録」をしてもらう**(古いキャッシュに古い番号が残っている可能性があるため)


---

## 修正・更新の手順(開発者向け)


**VS Codeで編集**

`index.html`, `manifest.json`, `service-worker.js` などをVS Codeで開いて修正する



**ローカルで動作確認**

VS Codeの拡張機能(Live Server など)や、ブラウザで直接ファイルを開いて表示を確認する



**GitHubに反映(push)**
```bash
git add .
git commit -m "修正内容の説明"
git push
```


**GitHub Pagesへの反映を待つ**

pushすると数十秒〜数分でGitHub Pagesに自動反映される



**必ず全員に再インストールを依頼する**

service-worker.jsのキャッシュ内容を変えた場合、古いキャッシュが端末に残っていることがある
ホーム画面から一度削除し、再度「ホーム画面に追加」をやり直してもらうと確実

## トラブルシューティング

| 症状 | 確認すること |
|---|---|
| 圏外でアプリが開かない | 事前に電波のある場所でホーム画面に追加(キャッシュ)できていたか確認 |
| アイコンが表示されない | manifest.json・service-worker.js内のファイル名/パスが一致しているか確認 |
| 圏外でSMSが送信できない | スマホが衛星通信対応機種か、空が見える屋外か、ソフトウェアが最新か確認 |
| 修正が反映されない | git pushできているか、古いキャッシュが端末に残っていないか確認 |
